# UI 框架

这是一个简单的 UI 框架，用于统一管理 UI 和 UI 事件

# 需求分析

  1. UI 窗口（Canvas）的统一管理（记录 / 隐藏所有窗口，提供查找窗口的方法）
  2. UI 事件管理

# 核心类

  1. UI 管理类 UIManager
    UIManager 用于管理（记录 / 隐藏）所有窗口，提供查找窗口的方法
  2. UI 窗口类 UIWindow
    UIWindow 是所有 UI 窗口的基类，可以代表所有窗口（概念继承，以层次化方式管理类）、定义所有窗口共有的行为（比如：显示和隐藏）
  3. UI 事件监听器 UIEventListener
    UIEventListener 提供所有能用到的 UI 事件（带事件参数类）
为什么要写UI事件监听器？

因为 Button 组件有几个致命的问题：

  • Button 只有单击事件，而其他大多数事件（光标按下、光标抬起...）都不具备。
  • Button 没有事件参数类（当多个按钮绑定同一个方法时，没有事件参数，就无法确定是哪个按钮被按下了）。
